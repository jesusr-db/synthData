Section 2 — Domain Coverage

Which of the 153 MVM tables get streaming vs. reference seeding vs. roadmap

Domain Strategy Key Tables Notes
Order Streaming guest_order, order_item, order_modifier, payment, status_event, delivery_order, channel Core heartbeat. Every 60s burst generates realistic order volume per location. Status events model the prep → ready → fulfilled lifecycle.
Inventory Streaming on_hand_balance, waste_log, receiving_order, replenishment_order, stock_transfer, adjustment on_hand_balance decrements with each order_item. Daily receiving orders restock PAR levels. Waste events ~3% of prep volume.
Guest Streaming guest_profile, digital_account, guest_preference ~40% of orders tied to a returning guest profile. New profiles created on ~15% of orders. Realistic repeat rate.
Loyalty Streaming member, loyalty_transaction, point_balance, reward_redemption ~30% of guest-linked orders earn points. ~8% of members redeem a reward per visit. Tier progression over time.
Workforce Streaming shift, labor_assignment, time_punch Shift schedules generated per location per day. Staffing levels scale with projected order volume. Occasional no-show events.
Restaurant Reference unit, location_profile, format_config, operating_hours, area_management, capacity_config, pos_terminal Seeded once. 100–500 units across US metros. District/region hierarchy (area_management). QSR format: carryout + delivery focus.
Menu Reference + slow updates menu_item, menu, recipe, item_price, nutrition_profile, allergen_declaration Seeded with Domino's-style catalog (~80 items: pizzas, wings, sides, drinks, desserts). LTO items added monthly. Price updates quarterly.
Finance Reference cost_center, profit_center, gl_account, financial_period GL structure seeded once. FK targets for order and inventory transactions. Financial periods auto-generated for the full date range.
Franchise Reference franchisee, territory, unit_ownership ~80% franchised units, ~20% corporate-owned. Realistic franchisee-to-unit ratios (avg 5–12 units per franchisee).
Procurement Reference + periodic supplier, purchase_order, goods_receipt Suppliers seeded once. Weekly purchase orders generated per location to match PAR replenishment. Receiving records match inventory receiving_order.
Food Safety Reference haccp_plan, critical_control_point, temperature_log HACCP plans seeded per menu category. Temperature logs generated periodically (4× per day per location). Occasional flagged events for realism.
unit_performance Computed unit_performance (AUV, COGS%, labor%, SSS) Derived by DLT Gold layer from order + inventory + workforce streams. No direct generation — computed from actuals.
Marketing Roadmap campaign, promotion, loyalty_program_config Complex campaign logic deferred to Phase 2. Promotions referenced as FK stubs only for now.
Real Estate Roadmap site, lease, property_profile Lease and property data not needed for core demo/analytics/ML use cases. Deferred.
Summary

5 domains fully streaming (Order, Inventory, Guest, Loyalty, Workforce) — these create the live "heartbeat."
6 domains reference/periodic (Restaurant, Menu, Finance, Franchise, Procurement, Food Safety) — seeded at deploy, updated slowly.
1 computed (unit_performance) — DLT Gold derives it from actuals.
2 deferred (Marketing, Real Estate) — Phase 2 roadmap.
