System Architecture — Section 1

Five components, fully deployable via Databricks Asset Bundle

External Signal (daily refresh)
API

Weather API (NOAA / OpenWeatherMap)

Per metro area. Called once at bootstrap (historical), then daily for 7-day forecast. Stored in ref.weather_conditions.

API

Events API (Ticketmaster / SeatGeek)

Sporting events, concerts, festivals per geo. Lat/lon matched to restaurant locations within 3 miles. Stored in ref.local_events.

Reference

US Holiday Calendar

Federal + regional holidays, Super Bowl, college football schedule. Hard-coded lookup table seeded at deploy time.

↓ demand multipliers
Event Generator (Databricks Job — runs every 60s for live, batch for backfill)
Spark Job

Synthetic Event Generator

Produces realistic events for each MVM domain. Two modes:
Backfill mode — generates N months of history in batch, timestamps replayed into the past.
Live mode — generates next 60 seconds of events, writes to staging with NOW() timestamps. Applies demand multipliers from weather, events, time-of-day curves, and day-of-week patterns.

Spark Job

Reference Seeder

Runs once at deploy. Creates 100–500 unit records across US metros, seeds menu items (Domino's-style: pizzas, wings, sides, drinks), staff, channels, franchisees, GL accounts, suppliers.

↓ raw events → staging Delta tables (append-only, partitioned by domain)

↓ readStream — DLT continuous pipeline
DLT Pipeline — Spark Declarative Pipeline (Silver + Gold)
DLT Silver

MVM Target Tables (153 tables)

Each MVM table declared as @dlt.table with readStream source, FK enrichment, and quality expectations (e.g., expect_or_drop("valid_order", "total_amount > 0")). Continuously updated as staging receives new events.

DLT Gold

Metric Views

Pre-built aggregations from metrics/ SQL files in the repo: AUV, COGS%, labor%, SOS compliance, loyalty redemption rate, waste %, SSS growth.
