Unity Catalog Layout

Single catalog, four schemas — clean separation of concerns

📦 catalog: qsr_synth

  📂 staging (Bronze — generator writes here)
    order_events
    inventory_events
    guest_events
    loyalty_events
    workforce_events

  📂 silver (MVM target — DLT writes here)
    order.guest_order, order.order_item …
    inventory.on_hand_balance …
    guest.profile, loyalty.member …
    all 153 MVM tables as sub-schemas

  📂 gold (Computed — DLT Gold layer)
    unit_performance_daily
    sos_compliance_summary
    loyalty_cohort_metrics
    inventory_waste_summary

  📂 ref (Seeded once — static/slowly changing)
    unit, menu_item, employee
    franchisee, supplier, gl_account …
    weather_conditions, local_events (Phase 2 stubs)

Job Topology & Dependencies

Execution order at deploy time, then steady-state continuous operation

Once at deploy
setup_job

Creates qsr_synth catalog + all schemas. Runs reference seeder (units, menu, staff, GL, franchisees, suppliers). Parameterized: num_units, backfill_months, catalog_name.

Once at deploy (batch)
generator_job — backfill mode

Generates N months of historical events for all domains. Writes to staging tables with replayed timestamps. Uses causal graph with hardcoded seasonality. Parallelized by location partition.

Continuous
DLT Pipeline — mvm_pipeline

Reads all staging tables via readStream. Writes all 153 MVM silver tables with quality expectations. Computes gold metric views. Runs in continuous mode with 30s trigger. Handles backfill and live stream in the same pipeline.

Every 60s (live)
generator_job — live mode

After backfill completes, switches to live mode. Generates 60s of events per tick using NOW() timestamps. Causal context evaluated per location per tick.

Manual trigger
destroy_job

Drops all staging and ref tables. Clears qsr_synth catalog. DAB-managed resources (pipeline, jobs) torn down via bundle destroy.

Phase 2 — Daily
daily_refresh_job

Calls NOAA + Ticketmaster APIs. Upserts ref.weather_conditions and ref.local_events. Causal graph picks them up automatically on next generator tick — no other changes needed.
